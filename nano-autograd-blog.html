02 核心概念：计算图与链式法则

自动微分的本质是将复杂的函数分解为一系列基本操作，并在这些操作之间建立一张有向无环图（DAG）。每次前向计算时，节点都会记录自己的"亲子关系"，为后续的梯度回传保留路径。

每条边都对应一次链式法则的应用：梯度从输出节点出发，逐层向输入节点流动，每经过一个操作节点就乘以该操作的局部导数。nano-autograd 所做的，正是自动化地构建和遍历这张图。

// 核心洞察
反向传播并不是什么神秘算法——它就是链式法则在计算图上的系统性应用，配合拓扑排序保证梯度流动的正确顺序。nano-autograd 让你在几十行代码内亲眼看到这一切。

03 Value 节点：引擎的基本单元

nano-autograd 的核心是一个名为 Value 的类。每个 Value 对象包裹一个标量数值，同时记录生成它的操作以及它的"父节点"——这是构成计算图的基本数据结构。

engine.py — Value 核心实现（精简展示）
class Value:
    def __init__(self, data, _children=(), _op=''):
        self.data  = data          # 前向计算值
        self.grad  = 0.0           # 梯度，初始为 0
        self._prev = set(_children)  # 父节点集合
        self._op   = _op          # 生成该节点的操作符
        self._backward = lambda: None  # 梯度回传函数（稍后填充）

    def __mul__(self, other):
        other = other if isinstance(other, Value) else Value(other)
        out = Value(self.data * other.data, (self, other), '*')

        def _backward():
            # 乘法的局部导数：d(a*b)/da = b，d(a*b)/db = a
            self.grad  += other.data * out.grad
            other.grad += self.data  * out.grad

        out._backward = _backward
        return out

    def relu(self):
        out = Value(0 if self.data < 0 else self.data, (self,), 'ReLU')

        def _backward():
            # ReLU 的梯度：正区间为 1，负区间为 0
            self.grad += (out.data > 0) * out.grad

        out._backward = _backward
        return out

注意 += 而非 = 的使用——这是梯度累积而非覆盖。当同一个节点被多条计算路径共享时（例如 x * x 中 x 同时是左操作数和右操作数），梯度必须从所有路径叠加，而不是只保留最后一条。这是实现正确反向传播的关键细节，许多教程都忽略了这一点。

04 反向传播：让梯度流动起来

有了计算图，反向传播的实现出人意料地简单：先对图做一次拓扑排序，确保每个节点的梯度在被"消费"前已经完整积累，然后按逆序调用每个节点的 _backward() 函数即可。

engine.py — backward() 实现
    def backward(self):
        # 第一步：拓扑排序，得到正确的计算顺序
        topo, visited = [], set()

        def build_topo(v):
            if v not in visited:
                visited.add(v)
                for child in v._prev:
                    build_topo(child)
                topo.append(v)

        build_topo(self)

        # 第二步：输出节点的梯度初始化为 1（dL/dL = 1）
        self.grad = 1.0

        # 第三步：逆序回传，链式法则自动传递
        for node in reversed(topo):
            node._backward()
// 为什么需要拓扑排序？
如果不保证顺序，一个节点的 _backward() 可能在其输出节点的梯度尚未完全到达之前就执行，导致梯度计算错误。拓扑排序保证了"下游先算完，上游再回传"这一不变式。

整个 backward() 方法不超过 15 行，却完整地实现了 PyTorch 引以为傲的自动微分核心。当你第一次亲手运行它、看到梯度值与手动推导完全吻合的那一刻，往往会有一种"原来如此"的醍醐灌顶感。

05 构建神经网络层

在 Value 引擎之上，nano-autograd 提供了一套轻量级的神经网络模块，包括 Neuron、Layer 和 MLP，接口风格与 PyTorch 高度一致。

nn.py — 神经元与多层感知机
class Neuron:
    def __init__(self, n_in, nonlin=True):
        # 权重用均匀分布初始化，打破对称性
        self.w = [Value(random.uniform(-1, 1)) for _ in range(n_in)]
        self.b = Value(0)
        self.nonlin = nonlin

    def __call__(self, x):
        # 线性组合 + 可选激活函数
        act = sum(wi * xi for wi, xi in zip(self.w, x)) + self.b
        return act.relu() if self.nonlin else act

    def parameters(self):
        return self.w + [self.b]


class MLP:
    def __init__(self, n_in, n_outs):
        # n_outs = [hidden_1, hidden_2, ..., output]
        sizes = [n_in] + n_outs
        self.layers = [
            Layer(sizes[i], sizes[i+1],
                  nonlin=(i != len(n_outs) - 1))  # 最后一层无激活
            for i in range(len(n_outs))
        ]

    def __call__(self, x):
        for layer in self.layers:
            x = layer(x)
        return x

值得注意的设计细节：输出层的神经元关闭了非线性激活（nonlin=False）。这是回归和分类任务的通用规范——分类任务在损失函数层处理激活（如 softmax），而不是在网络层内部硬编码。nano-autograd 的实现体现了这一正确的工程直觉。

07 动手实验：训练一个二分类 MLP

以下是一个完整的端到端训练示例，展示了 nano-autograd 的完整工作流——从构建网络、定义损失、执行反向传播到参数更新：

train_example.py — 完整训练循环
from nano_autograd.engine import Value
from nano_autograd.nn     import MLP

# ── 1. 构建网络：2 输入 → 16 → 16 → 1 输出 ──
model = MLP(2, [16, 16, 1])
print(f"参数量：{len(model.parameters())}")  # → 337

# ── 2. 准备数据（XOR 问题）──
xs = [[0,0], [0,1], [1,0], [1,1]]
ys = [-1,  1,  1, -1]  # 标签：+1 / -1

# ── 3. 训练循环 ──
for step in range(100):

    # 前向传播
    ypred = [model(x) for x in xs]

    # Hinge Loss（SVM 风格）
    loss = sum(
        (1 - yi * yout).relu()
        for yi, yout in zip(ys, ypred)
    )

    # 反向传播前清零梯度（否则会累积）
    for p in model.parameters():
        p.grad = 0.0

    # 反向传播
    loss.backward()

    # SGD 参数更新
    lr = 0.05
    for p in model.parameters():
        p.data -= lr * p.grad

    if step % 10 == 0:
        print(f"step {step:3d} | loss = {loss.data:.4f}")

这段代码完整地呈现了深度学习训练循环的六个核心步骤：构建模型 → 前向传播 → 计算损失 → 梯度清零 → 反向传播 → 参数更新。其中梯度清零是初学者最常遗忘的一步——nano-autograd 的显式梯度管理让这一陷阱一览无余，从而帮助你建立正确的认知模型。
